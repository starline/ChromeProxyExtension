# Proxy Injector V3

Расширение для **Google Chrome** (Manifest V3), которое переключает трафик браузера через **HTTP-прокси** с фиксированным портом **3128**. Адрес прокси-сервера задаётся в настройках.

## Возможности

- **Включение / выключение прокси** одним кликом по иконке расширения на панели инструментов.
- **Настраиваемый хост** прокси (порт зашит в коде: `3128`).
- **Автозапуск**: опция «включать прокси при старте браузера».
- **Индикатор на иконке**: при включённом прокси бейдж показывает `ON` после успешной проверки доступности через `https://www.google.com/generate_204`, иначе `OFF`.
- **Обход локальных адресов**: в списке исключений указано `<local>`.

## Установка (режим разработчика)

1. Откройте `chrome://extensions/`.
2. Включите **Режим разработчика**.
3. Нажмите **Загрузить распакованное расширение** и выберите папку с этим проектом.

## Настройка

1. Откройте страницу настроек расширения (правый клик по иконке → **Параметры** или через список расширений).
2. Укажите **Proxy Host** (имя или IP сервера без порта).
3. При необходимости включите **Enable proxy on browser startup**.
4. Нажмите **Save**.

Если хост не задан, расширение не включает прокси и оставляет прямое подключение.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `manifest.json` | Манифест MV3, права `proxy` и `storage`, service worker |
| `background.js` | Логика прокси, переключение по клику, проверка соединения |
| `options.html` / `options.js` | Страница настроек хоста и автозапуска |

## Требования

- Chromium-браузер с поддержкой Manifest V3 и API `chrome.proxy`.

## Примечание

Порт прокси **3128** и схема **http** заданы в `background.js` в функции `buildProxyConfig`. При смене сервера на другой порт или тип прокси нужно править код.
